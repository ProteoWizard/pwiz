# PRM tutorial -- SkylineMCP / AI Connector steps
#
# Each line is one operation: a verb name, then its arguments.
# Lines starting with '#' and blank lines are ignored.
#
# Argument format:
#   - formId and controlId arguments are wrapped in double quotes.
#   - The VALUE argument of SetFormValue is the rest of the line after the controlId
#     (taken verbatim, so it may itself contain spaces and double quotes -- needed for
#     multi-file selection, see below).
#
# Verbs that map directly to IJsonToolService methods:
#   InvokeMenuItem  "<menu path>"                     -> skyline_invoke_menu_item
#   ClickFormButton "<formId>" "<button>"             -> skyline_click_form_button
#   SetFormValue    "<formId>" "<controlId>" <value>  -> skyline_set_form_value
#
# Control matching (ClickFormButton / SetFormValue controlId):
#   - A control is matched by its control Name OR its visible Text. Matching prefers an exact
#     match, then a match after stripping non-alphanumeric symbols, and among equal matches a
#     visible+enabled control -- so the tutorial's "Next" matches the "Next >" button and
#     "Finish" the (visible) Finish button, even though a hidden early-Finish shares the caption.
#   - SetFormValue sets: a TextBox's text; a CheckBox/RadioButton checked state
#     ("true"/"false"); or a ComboBox selection (by visible item text).
#   - A text field labelled only by an adjacent Label (e.g. "Ion match tolerance") cannot
#     yet be addressed by that label -- only by its own control Name. The steps below avoid
#     this (the tutorial does not need to change those fields on the streamlined path).
#
# Harness synchronization primitives (poll skyline_get_open_forms; NOT MCP methods):
#   WaitForForm     "<formId>"   -- wait until that form/dialog is listed by GetOpenForms
#   WaitForNoForm   "<formId>"   -- wait until it is gone
# Required because InvokeMenuItem / ClickFormButton return immediately (the dialog-watch
# makes them return as soon as a dialog appears), so the resulting form is usually not
# open yet on the next line. Page-to-page moves WITHIN the wizard reuse the same formId
# ("ImportPeptideSearchDlg:Import Peptide Search"), so there is no new form to wait for
# between consecutive Next clicks; the dialog-watch makes each click return only after
# its synchronous work (including the library build and the final import) completes.
#
# formIds are the stable "TypeName:Title" identifiers from GetOpenForms; they are
# deterministic for this tutorial, so no run-time inspection/branching is needed --
# GetOpenForms is used only for the Wait* steps. (The titles below are the English UI
# strings; a localized run needs the localized titles.)
#
# {DATA} is replaced by the test with the PRM tutorial data directory (the folder that
# contains BSA_Protea_label_free_meth3.sky and the 'search' subfolder). This was validated
# against the mzML download (TargetedMSMSMzml_2.zip); the .raw download works the same.
#
# Validation: every step below was executed end to end against Skyline-daily over the pipe,
# from File > Open through the completed chromatogram import (document ends with 2
# replicates, 20fmol and 80fmol). The later "Reviewing the Data" section of the tutorial is
# NOT captured here: its View-menu steps are InvokeMenuItem-able, but most of its remaining
# steps use right-click GRAPH CONTEXT MENUS (Normalize To, Show Library, Transitions > Split
# Graph, ...), which the connector cannot drive yet (InvokeMenuItem is main-menu only).

# --- (Optional) Get past the Start Page -------------------------------------------------
# Only needed when Skyline launches to the Start Page (Settings.Default.ShowStartupForm).
# A test that disables the start page can skip these two lines and start at File > Open.
# ClickFormButton works on the Start Page's tiles because ActionBoxControl is an
# IButtonControl whose Caption is its visible label.
# WaitForForm     "StartPage:Start Page"
# ClickFormButton "StartPage:Start Page" "Blank Document"
# WaitForNoForm   "StartPage:Start Page"

# --- Open the starting document ---
InvokeMenuItem  "File > Open"
WaitForForm     "FileDialog:Open"
SetFormValue    "FileDialog:Open" "FileName" {DATA}\BSA_Protea_label_free_meth3.sky
ClickFormButton "FileDialog:Open" "Open"
WaitForNoForm   "FileDialog:Open"

# --- File > Import > Peptide Search (document is saved, so no save prompt) ---
InvokeMenuItem  "File > Import > Peptide Search"
WaitForForm     "ImportPeptideSearchDlg:Import Peptide Search"

# Spectral Library page: keep only spectra that identify document peptides, and pick the PRM
# workflow (selecting the radio gives the PRM full-scan defaults on the later page).
SetFormValue    "ImportPeptideSearchDlg:Import Peptide Search" "Filter for document peptides" true
SetFormValue    "ImportPeptideSearchDlg:Import Peptide Search" "PRM" true

# --- Add both search-result files at once ---
# Multi-select on the common (classic) Open dialog: navigate INTO the folder first, then
# set the file name to the QUOTED BARE filenames separated by spaces. (Setting quoted
# FULL paths is rejected with "Operation cannot be performed"; bare names in the current
# folder work.)
ClickFormButton "ImportPeptideSearchDlg:Import Peptide Search" "Add Files"
WaitForForm     "FileDialog:Add Input Files"
SetFormValue    "FileDialog:Add Input Files" "FileName" {DATA}\search
ClickFormButton "FileDialog:Add Input Files" "Open"
SetFormValue    "FileDialog:Add Input Files" "FileName" "klc_20100329v_Protea_Peptide_Curve_20fmol_uL_tech1.perc.xml" "klc_20100329v_Protea_Peptide_Curve_80fmol_uL_tech1.perc.xml"
ClickFormButton "FileDialog:Add Input Files" "Open"
WaitForNoForm   "FileDialog:Add Input Files"

# Both .perc.xml files are now in the wizard's grid. Their Score Type is "Percolator q-value"
# and the Score Threshold defaults to 0.01 -- the tutorial's value -- so no SetFormValue is
# needed for the threshold.

# --- Build the spectral library (Next) ---
# This runs BiblioSpec, then advances the wizard. With "Filter for document peptides" on, a
# "Skyline" information dialog (MessageDlg) lists peptides excluded for ambiguous matches and
# must be dismissed. (Data-dependent: on data with no ambiguous matches this dialog does not
# appear -- skip the next three lines if so.)
ClickFormButton "ImportPeptideSearchDlg:Import Peptide Search" "Next"
WaitForForm     "MessageDlg:Skyline"
ClickFormButton "MessageDlg:Skyline" "OK"
WaitForNoForm   "MessageDlg:Skyline"

# --- Extract Chromatograms page: Skyline found the two mzML/raw files. Advance. ---
ClickFormButton "ImportPeptideSearchDlg:Import Peptide Search" "Next"

# Skyline detects a common prefix/suffix on the long file names and offers to shorten them.
WaitForForm     "ImportResultsNameDlg:Import Results"
ClickFormButton "ImportResultsNameDlg:Import Results" "OK"
WaitForNoForm   "ImportResultsNameDlg:Import Results"

# --- Add Modifications page (no modifications to add for these peptides). Advance. ---
ClickFormButton "ImportPeptideSearchDlg:Import Peptide Search" "Next"

# --- Configure Transition Settings page (defaults are fine). Advance. ---
ClickFormButton "ImportPeptideSearchDlg:Import Peptide Search" "Next"

# --- Configure Full-Scan Settings page (PRM defaults from the workflow choice). Finish. ---
# On the last page the Next button reads "Finish"; clicking it closes the wizard and starts the
# chromatogram import. The dialog-watch waits through the "Loading chromatograms" progress, so
# this click returns only once the wizard has closed and import has begun.
ClickFormButton "ImportPeptideSearchDlg:Import Peptide Search" "Finish"
WaitForNoForm   "ImportPeptideSearchDlg:Import Peptide Search"

# The document now has 2 replicates (20fmol, 80fmol) with extracted chromatograms.

# --- Reviewing the Data (PARTIAL -- main-menu steps only) -------------------------------
# Validated live. The tutorial interleaves these with right-click GRAPH CONTEXT MENUS (Show
# Dot Product > Labels, Normalize To > None, Show Library, Transitions > Split Graph, ...),
# which are NOT yet driveable (InvokeMenuItem is main-menu only) -- so this section stops where
# the tutorial's first context-menu step would come.
InvokeMenuItem  "View > Arrange Graphs > Tiled"
InvokeMenuItem  "Edit > Collapse All > Precursors"
InvokeMenuItem  "View > Auto-Zoom > Best Peak"
InvokeMenuItem  "View > Peak Areas > Replicate Comparison"
InvokeMenuItem  "View > Transitions > Products"

# --- Notes / known limitations found while validating against a live Skyline ---
# 1. The native Save As / Save dialog IS now driveable (it is the modern IFileDialog: file name
#    set via WM_SETTEXT on its Edit, accepted via the Save button). It lists as
#    "FileDialog:Save As". This tutorial opens an already-saved document, so it is not exercised
#    here; saving would be SetFormValue "FileDialog:Save As" "FileName" <path> + ClickFormButton.
# 2. Still NOT supported: a graph right-click context-menu item (the main remaining gap for the
#    review section), selecting a tabbed settings dialog's tab (SelectTab), and checking a
#    precursor in a tree pop-up pick-list. The streamlined path above avoids the latter two by
#    configuring full-scan inside the wizard (no tabs) instead of via Settings > Transition Settings.
# 3. RadioButton selection (e.g. the "PRM" workflow, the "Retention time filtering" options)
#    is done with SetFormValue "<formId>" "<radio label>" true.
