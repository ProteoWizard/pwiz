# InvokeMenuItem, InvokeContextMenuItem, ClickFormButton, SetFormValue, and the WaitForForm /
# WaitForNoForm sync primitives). Each localized tutorial needs its own; {DATA} is not localized.

# {DATA} is the LiveReports folder that contains Rat_plasma.sky and SampleInfo.txt
# (https://skyline.ms/tutorials/LiveReports.zip).

# STATUS: this tutorial is much more dialog- and grid-heavy than PRM, and exercises several verbs
# the connector does NOT have yet. Only the opening steps below are validated; the rest of the

# can continue incrementally as those verbs land.

# --- Open the starting document ---
# (If a prior document has unsaved changes, File > Open first shows a "Save changes?" prompt
# (MultiButtonMsgDlg) -- answer No to discard. Validated.)
InvokeMenuItem  "File > Open"
WaitForForm     "FileDialog:Open"
SetFormValue    "FileDialog:Open" "FileName" {DATA}\Rat_plasma.sky
ClickFormButton "FileDialog:Open" "Open"
WaitForNoForm   "FileDialog:Open"
# Document: 48 proteins, 125 peptides, 721 transitions, 42 replicates.

# --- Enable Audit Logging ---
# View > Live Reports > Audit Log opens the Audit Log form. Validated.
InvokeMenuItem  "View > Live Reports > Audit Log"
WaitForForm     "AuditLogForm:Audit Log: All Info"
# Check the "Enable audit logging" checkbox. This CheckBox has AutoCheck=false and acts on its
# Click handler, so it must be CLICKED (ClickFormButton), not value-set (SetFormValue would only set
# Checked and not run the handler). ClickFormButton now clicks checkboxes -- validated.
ClickFormButton "AuditLogForm:Audit Log: All Info" "Enable audit logging"
# NEXT STEP NOT YET DRIVEABLE: "Close the Audit Log window" -- no close-form verb. See gap (2).

# --- Showing the Document Grid ---
InvokeMenuItem  "View > Live Reports > Document Grid"
WaitForForm     "DocumentGridForm:Document Grid: Proteins"
# The "Reports" dropdown BUTTON (a ToolStripDropDownButton) is now clickable -- validated:
ClickFormButton "DocumentGridForm:Document Grid: Proteins" "Reports"
# ... but its ITEMS (Proteins / Peptides / Replicates / Customize Report / ...) are built
# dynamically when the dropdown opens and are not in the static DropDownItems, so clicking an item
# by name does not work yet. See gap (1b).
