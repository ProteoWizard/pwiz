Osprey Model Diagnostics

Trained scoring model — feature contributions

The averaged cross-validated linear (Percolator/SVM) model, decomposed Skyline-style into each feature's percent contribution to the composite target–decoy separation. Rows with a negative percent contribution (the feature pulls decoys up) are shown in red.

Composite score distribution

Best-per-precursor composite score for each class, with a normal curve fit to the decoy scores — the Osprey analog of Skyline's mProphet "Model Scores" view, extended with entrapment.

Score density by class (area-normalized)

The same best-per-precursor scores as probability densities, so the shapes compare regardless of class size. If entrapment (p_target) tracks the decoys, the generated/library decoys are faithful nulls; if it rides higher toward the targets, the decoys understate the true error.

FDR calibration — reported q vs true FDP

Entrapment-measured true false-discovery proportion versus Osprey's reported q-value. A perfectly calibrated q sits on the dashed y = x line. This is the FDRBench measurement, computed in-process from the pairing manifest — no external FDRBench run required.

Identification yield

Accepted real-target precursors as the reported q-value threshold sweeps.

Paired decoy-win fraction

Within each target–decoy pair (shared base_id), how often the decoy outscores its target, binned by the winner's score. A fair null sits at the 50% line. Real pairs dipping below 50% at high score is native signal winning as it should; entrapment pairs holding the coin confirms the decoys are honest nulls. Marginal densities cannot see this within-pair ordering — only the competition can.
