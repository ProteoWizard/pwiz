Each feature's standardized target vs decoy distribution (the interactive report shows one at a time on click; printed here in full).

Score density by class (area-normalized)

The same best-per-precursor scores as probability densities, so the shapes compare regardless of class size. If entrapment (p_target) tracks the decoys, the generated/library decoys are faithful nulls; if it rides higher toward the targets, the decoys understate the true error.

Null-alignment density ratio (non-parametric)

The per-class score densities above, divided: target : decoy (the diagnostic) and, when entrapment is present, p_target : p_decoy (a matched-null reference). On a fair (equal-chance) pairing the target:decoy line rides a flat plateau across the null-dominated left side, then rises where real hits begin; a sloping left side means the decoys don't track the false-target null — an equal-chance / decoy-quality violation, visible with no parametric fit (unlike the decoy-normal overlay, which misfits skewed decoys). Storey's original check.

FDR calibration — Osprey q vs true FDP

Entrapment-measured true false-discovery proportion versus Osprey's own q-value, computed in-process from the pairing manifest — no FDRBench install. Pick a q-axis: the experiment-wide views reproduce the FDRBench --fdrbench plots exactly (FDRBench just counts entrapment on the q Osprey emits); the per-run views are a picture FDRBench can't show. Each view has a [0, 2%] zoom (FDP axis auto-scaled) and a full-extent panel.

Identification yield

Accepted real-target precursors as the reported q-value threshold sweeps.

Paired decoy-win fraction

Within each target–decoy pair (shared base_id), how often the decoy outscores its target, binned by the winner's score. A fair null sits at the 50% line. Real pairs dipping below 50% at high score is native signal winning as it should; entrapment pairs holding the coin confirms the decoys are honest nulls. Marginal densities cannot see this within-pair ordering — only the competition can.
