pwiz-sharp Sciex reference override
====================================

Reference mzMLs in this directory take precedence over the canonical cpp fixtures
at `pwiz/data/vendor_readers/ABI/Reader_ABI_Test.data/` when
`VendorReaderTestHarness` resolves the diff target. The lookup order is:

1. `<test-assembly-bin>/Reference/<filename>` (this directory at runtime)
2. `<rootPath>/<filename>` (cpp tree)

**Why a separate directory?** Editing files under `pwiz/data/vendor_readers/ABI/`
retriggers every C++ vendor TeamCity config because the cpp test harness reads
from that tree. Pinning pwiz-sharp-specific reference output here keeps the TC
blast radius limited to pwiz-sharp's own configs.

**Current contents:**
- `50uMpyrone-8uL-01-024-simSpectra.mzML` — generated by cpp `msconvert` against
  `50uMpyrone-8uL-01.wiff` with `--simAsSpectra --filter "index 0-100"`. The .wiff
  fixture itself stays in the cpp tree (cpp's `Reader_ABI_Test` references it by
  the same path); only the reference output lives here.

**To regenerate**: run cpp `msconvert` against the .wiff with the test config
(`--simAsSpectra --filter "index 0-100"`, `--64 --zlib` to match pwiz-sharp's
default encoding) and overwrite the file here. Don't touch the cpp tree.

**To remove**: when the cpp reference gets regenerated to match pwiz-sharp's
output (or vice versa), delete the file here so the harness falls back to the
cpp copy.
